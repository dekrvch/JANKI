# JANKI - Japanese Flashcard App

A modern Japanese language learning application with spaced repetition system (SRS), built with Vite, TypeScript, and Firebase.

## Features

- **Spaced Repetition System (SRS)**: Optimized learning schedule with increasing intervals (1, 3, 7, 14, 30, 60, 90 days)
- **Bidirectional Practice**: Practice both English-to-Japanese (EJ) and Japanese-to-English (JE)
- **Custom Hiragana Keyboard**: Interactive keyboard with support for:
  - Single tap: Add character
  - Double tap: Add voiced marks (゛゜)
  - Long press: Add small characters (ゃゅょっ etc.)
- **Automatic Audio Generation**: Google Cloud TTS generates Japanese pronunciation automatically
- **Progress Tracking**: Track learning progress per card and direction
- **Google Authentication**: Secure sign-in with Google
- **Import/Export**: JSON-based card import/export functionality

## Tech Stack

### Frontend
- **Vite**: Fast build tool and dev server
- **TypeScript**: Type-safe JavaScript
- **Firebase SDK v10**: Modular Firebase integration
  - Authentication (Google OAuth)
  - Firestore (card storage)
  - Cloud Storage (audio files)

### Backend
- **Firebase Cloud Functions**: Serverless functions
- **Google Cloud Text-to-Speech**: Automatic audio generation
- **Firebase Hosting**: Static site hosting

## Setup Instructions

### Prerequisites

- Node.js 18+
- Firebase CLI: `npm install -g firebase-tools`
- Firebase project (already configured as `yanki-card`)

### 1. Install Dependencies

```bash
npm install
```

### 2. Install Functions Dependencies

```bash
cd functions
npm install
cd ..
```

### 3. Environment Variables

The `.env` file is already created with Firebase configuration. For production deployment, add these secrets to GitHub:

- `VITE_FIREBASE_API_KEY`
- `VITE_FIREBASE_AUTH_DOMAIN`
- `VITE_FIREBASE_PROJECT_ID`
- `VITE_FIREBASE_STORAGE_BUCKET`
- `VITE_FIREBASE_MESSAGING_SENDER_ID`
- `VITE_FIREBASE_APP_ID`

### 4. Google Cloud TTS Setup

For audio generation, you need to:

1. Enable Google Cloud Text-to-Speech API in your Firebase project
2. The Cloud Function will use Firebase's default service account

## Security Rules

### Firestore Rules
- Users can only read/write their own cards (`/users/{userId}/cards/*`)
- Card data is validated on creation
- Cloud Functions can update cards (for audio URLs)

### Storage Rules
- Users can read their own audio files
- Only Cloud Functions can write audio files (prevents abuse)

## Card Data Structure

### Card Document (Firestore)
```typescript
{
  japanese: string;      // "猫"
  reading: string;       // "ねこ"
  english: string;       // "cat"
  audioUrl: string;      // Generated by Cloud Function
  createdAt: Timestamp;
  progress: {
    ej: {                // English-to-Japanese
      repsCorrect: number;
      learned: boolean;
      interval: number;  // Days until next review
      nextDue: string;   // YYYY-MM-DD
    },
    je: {                // Japanese-to-English
      repsCorrect: number;
      learned: boolean;
      interval: number;
      nextDue: string;
    }
  }
}
```

### Import Format (JSON)
```json
[
  {
    "japanese": "猫",
    "reading": "ねこ",
    "english": "cat"
  },
  {
    "japanese": "犬",
    "reading": "いぬ",
    "english": "dog"
  }
]
```

## How It Works

### 1. User Flow
1. Sign in with Google
2. Import cards via JSON or add manually
3. Start a session (combines due reviews + new cards)
4. Answer cards using hiragana keyboard or text input
5. Grade answers (correct/wrong)
6. Progress is automatically saved to Firestore

### 2. Audio Generation
When a card is created or the Japanese text is updated:
1. Firestore trigger detects the change
2. Cloud Function calls Google Cloud TTS
3. Audio file is generated and uploaded to Storage
4. Card is updated with `audioUrl`

### 3. Spaced Repetition
- **New cards**: Require 3 correct answers to graduate
- **Learned cards**: Follow interval sequence [1, 3, 7, 14, 30, 60, 90] days
- **Incorrect reviews**: Reset to 1-day interval

## Migration Notes

- **Before**: Single-file HTML app with Firebase Realtime Database
- **After**: Modular Vite + TypeScript with Firestore

- Firebase Realtime Database → Firestore
- Inline JavaScript → Modular TypeScript
- No build system → Vite
- Manual audio upload → Automatic generation via Cloud Functions
- No security rules → User-specific rules

## Troubleshooting

### Build fails with "Property 'env' does not exist"
- Make sure `src/vite-env.d.ts` exists
- Restart TypeScript server in your IDE

### Audio generation not working
- Verify Google Cloud TTS API is enabled
- Check Cloud Functions logs: `firebase functions:log`
- Ensure Storage rules allow Cloud Function writes

### Authentication fails
- Verify Firebase Auth is enabled with Google provider
- Check that `authDomain` in config matches your Firebase project
